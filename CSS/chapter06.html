Advance CSS

CSS variables (custom properties) are a modern tool to manage, reuse, and dynamically update style values across your web project. With these, you can enable easy theming, simplify maintenance, and make style adjustments with minimal code changes.

1. Introduction to CSS Variables

CSS variables, called custom properties, use the --property-name syntax and are reusable throughout your stylesheet. They enable dynamic and scalable design systems easily handled in modern web projects.

2. Syntax & Usage

  • :root { --primary-color: #3498db; } (Global variable definition)
  • .container { background: var(--primary-color); } (Using variable)
  • var(--property, fallback) (Optional fallback for robustness)

3. Global vs. Local Scope

Define variables globally with :root for entire project access, or locally in a class for component-specific customization.

  • :root { --border-radius: 8px; } (Global)
  • .card { --border-color: #ddd; } (Local to .card)

4. Dynamic Theming

Easily implement light or dark modes by overriding variables, reducing duplication. For example:

  • :root { --bg: #fff; --text: #000; } (Default theme)
  • .dark-mode { --bg: #000; --text: #fff; } (Dark theme override)
  • body { background: var(--bg); color: var(--text); } (Usage)

5. Animation Control

Centralize your animation timing:

  • :root { --transition-speed: 0.3s; }
  • .button { transition: background var(--transition-speed); }

6. Best Practices

  • Define all global variables in :root for easy access.
  • Use semantic names (e.g., --btn-bg).
  • Override locally only when needed for components/themes.
  • Keep variable structure flat, avoid deep inheritance unless necessary.

7. CSS Variables vs. Preprocessor Variables

Property CSS Variables Sass/Less Variables
Syntax --property-name $variable / @variable
Scope DOM Cascade (dynamic) File-scope (static)
Dynamic Change Possible via JS, live update Requires recompile
Inheritance Yes, via cascade No
