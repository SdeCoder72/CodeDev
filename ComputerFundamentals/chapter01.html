1. Binary Representation of Unsigned Numbers

Unsigned binary representation is a way to represent positive numbers using only 0s and 1s. All bits contribute to magnitude, with no sign bit.

Features of Unsigned Representation:

  • Each bit represents a power of 2.
  • Rightmost bit (LSB), leftmost bit (MSB).
  • Value is sum of powers of 2 at each '1'.
  • An n-bit unsigned binary: range 0 to 2ⁿ - 1.

Limitations:

  • Represents only positive values and zero.
  • Limited by number of bits (e.g., 0-255 for 8 bits).
  • Overflow errors beyond max value.

2. Binary Representation of Signed Numbers

Allows representation of both positive and negative integers. Common methods:

Sign-Magnitude

MSB is the sign (0=positive, 1=negative). Remaining bits for magnitude.

Limitations:

  • Two zero representations (positive/negative zero).
  • Arithmetic operations are complex.
  • Range is not symmetric.

One's Complement

Negative numbers found by flipping all bits of the positive number.

Limitations:

  • Separate +0 and -0 representations.
  • Needs end-around carry for addition.

Two's Complement

Invert all bits and add 1 for negative numbers. Used in almost all computers today.

Limitations:

  • Asymmetric range (one extra negative value).
  • Requires sign check for conversion.

3. Floating-Point Representation (IEEE 754)

Used for representing real numbers using sign, exponent, and mantissa.

32-bit Single Precision Format:

Features:

  • Uses 1 bit for sign, 8 bits for exponent, and 23 bits for mantissa.
  • Follows IEEE 754 standard for floating-point representation.
  • Efficient for storing real numbers with moderate precision.

Limitations:

  • Limited precision due to smaller mantissa.
  • Can cause rounding errors in calculations.

64-bit Double Precision Format:

Features:

  • Can represent very large and small values.
  • Precision via mantissa, range via exponent.
  • Uses 1 bit for sign, 11 bits for exponent, and 52 bits for mantissa.

Limitations:

  • Rounding errors.
  • Cannot represent all decimals exactly.
  • Consumes more memory than single precision.

4. Gray Code Binary Representation

Each successive value differs by only one bit. Useful in encoders and error-reduction systems.

Features:

  • Reduces transition errors.
  • Simple bitwise conversion rules.

Limitations:

  • Arithmetic is complex.

5. Binary-Coded Decimal (BCD) Representation

Each decimal digit is represented by its own 4-bit binary value. Used in digital clocks, calculators, etc.

Features:

  • Each decimal digit is represented using 4-bit binary.
  • Easy conversion between binary and human-readable decimal.
  • Useful in financial and digital display systems.

Limitations:

  • Inefficient storage—wastes bits compared to pure binary.
  • Arithmetic operations are more complex.

6. Excess-3 Code Representation

Decimal digits encoded as 4-bit binary values with +3 offset. Simplifies some arithmetic and error detection.

Features:

  • Avoids negative digit codes.
  • Useful for some error detection.

Limitations:

  • Space inefficient for large numbers.
  • Limited use compared to other codes.
