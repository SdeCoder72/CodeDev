File Systems in Operating System

File systems are the backbone of data storage, organizing information and making it easily retrievable for users and applications.

1. What is a File System?

A file system is the method and data structure that an operating system uses to control how data is stored and retrieved. Without a file system, data would be essentially a large block of raw information with no way to tell where one piece stops and another begins.

Example File Systems:

  • NTFS, FAT32 (Windows)
  • EXT4 (Linux)
  • HFS+ (macOS)

2. Functions of File System

  • Organizes files and directories.
  • Manages file access, permissions, and security.
  • Handles space allocation and free space management.
  • Supports file creation, deletion, modification, and backup.

3. Types of File Systems

Various file systems exist, each with unique features and limitations.

  • Single-Level Directory
  • Two-Level Directory
  • Tree-Structured Directory
  • Acyclic Graph Directory
  • General Graph Directory

4. Directory Structures

Directory structures determine how files are organized and navigated:

  • Single-Level: All files in one directory.
  • Hierarchical: Files grouped in subdirectories.
  • Graph: Allows links between files/directories.

5. File Allocation Methods

Three primary ways files are stored on disk:

  • Contiguous Allocation: Files occupy a set of contiguous disk blocks.
  • Linked Allocation: Each file block points to the next.
  • Indexed Allocation: Index block contains pointers to all file blocks.

6. Features & Limitations

Features:

  • Efficient file management
  • Security and data integrity
  • Support for different data types (text, binary, images, etc.)

Limitations:

  • Fragmentation issues (depends on allocation method)
  • Space inefficiency for small files
  • Complex recovery in case of failure
